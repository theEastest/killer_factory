# 加工台逻辑重构任务

## 一、删除旧版加工台结构

删除基础加工台原有的以下槽位：

* 工件槽；
* 投料缓存槽。

删除旧版“登记固定工件并连续加工”的逻辑。

重构后，基础加工台只保留：

* **一个工序缓存槽**。

加工台本身不长期保存待加工构件，也不长期缓存材料。

---

# 二、新版加工台结构

基础加工台具有：

* 一个工序缓存槽；
* 当前机械电量；
* 启动按钮；
* 工序牌缩略图；
* 当前工序所需材料提示；
* 启动失败原因提示。

玩家可以在自己的回合内，且动作队列为空时：

* 将任意带有“工序”关键词的牌拖入工序缓存槽；
* 将当前工序牌取回手牌；
* 用另一张工序牌替换当前工序牌。

上述操作：

* 不消耗玩家能量；
* 不消耗机械电量；
* 不视为打出工序牌；
* 不触发工序牌的消耗；
* 不触发工序牌的主体效果；
* 手牌已满时不能取回或替换工序牌。

---

# 三、加工台材料来源

加工台不使用独立的投料槽。

加工台启动时，根据当前缓存工序的材料要求，自动从以下区域寻找材料：

1. 玩家手牌；
2. 仓储室及其他具有仓储功能的机械缓存。

基础加工台不得从以下位置自动取料：

* 机械臂中装填的卡牌；
* 分解室缓存；
* 其他非仓储类机械缓存；
* 抽牌堆；
* 弃牌堆；
* 消耗区。

## 材料选择优先级

自动选择材料时使用固定顺序：

1. 优先使用工序要求的专用原料；
2. 专用原料不足时，使用万能原料替代；
3. 优先使用手牌中最左侧的合法材料；
4. 然后按照机械从左到右的顺序，从仓储室取出合法材料；
5. 同一仓储室中按照库存顺序消耗。

必须保证自动材料选择具有确定性，不得随机选择材料。

## 多材料工序

若某个工序需要多张材料，只有在全部材料都能支付时才允许启动。

例如工序需要：

* 两张万能原料；

但当前只有一张万能原料，则加工台不能启动，也不能先消耗其中一张。

# 四、新版加工台启动流程

玩家单击加工台的“启动”按钮后，按照以下顺序处理。

## 第一步：检查机械状态

检查：

* 当前是否处于玩家回合；
* 动作队列是否允许打开选择界面；
* 加工台是否正在执行其他动作；
* 工序缓存槽中是否存在工序牌。

任意条件不满足时，启动失败并播报原因。

## 第二步：验证工序牌

检查缓存卡牌：

* 是否具有“工序”关键词；
* 当前工序是否能够通过加工台执行；
* 工序数据是否完整；
* 工序是否定义了合法目标规则和材料规则。

工序无效时，启动失败。

## 第三步：检查机械电量

检查加工台是否至少拥有1点可用机械电量。

可用电量仍按照全局机械供能顺序判定：

1. 手牌中的便携电池；
2. 储能室；
3. 加工台自身电量。

电量不足时，启动失败。

此阶段只检查，不实际支付电量。

## 第四步：检查材料

按照当前工序的材料需求，在手牌和仓储机械中查找合法材料。

若材料不足，启动失败。

此阶段只检查和记录候选材料，不实际消耗。

## 第五步：检查合法构件

从玩家当前手牌中查找能够接受该工序的构件。

目标只能来自实际手牌，不能来自：

* 机械臂；
* 仓储室；
* 分解室；
* 其他机械缓存；
* 抽牌堆；
* 弃牌堆。

合法性必须根据具体工序判断，例如：

* 固化只能选择一次性构件；
* 标准升级不能选择已经升级的构件；
* 精密化不能对已经拥有精密构件且没有脆弱构件的目标重复执行；
* 脆弱构件第二次接受加工时，会进入损坏流程，但仍可显示为危险目标；
* 外加弹簧不能对已经拥有外加弹簧的构件重复生效；
* 轻量化可以选择费用大于0的构件；
* 标准校准必须选择至少有一个可强化伤害或格挡数值的构件；
* 手工装配需要额外检查作为融锻材料的构件。

若手牌中没有任何合法构件，加工台启动失败。

## 第六步：弹出构件选择页面

前五项检查全部通过后，弹出手牌构件选择页面。

选择页面：

* 只显示当前手牌中的合法构件；
* 非法卡牌隐藏或置灰；
* 每张卡显示当前完整卡面；
* 显示该工序作用后的结果预览；
* 允许玩家取消；
* 取消不消耗电量或材料；
* 选择界面打开期间暂停其他产线自动动作进入队列。

即使只有一个合法构件，也应显示选择页面，不自动替玩家选择。

## 第七步：重新验证

玩家确认目标后，在真正创建加工动作前重新验证：

* 工序牌仍在加工台中；
* 目标构件仍在手牌中；
* 目标仍然合法；
* 材料仍然足够；
* 电量仍然足够；
* 加工台未被其他动作占用。

若状态在选择期间发生变化，则取消本次启动并播报具体原因。

## 第八步：支付资源并执行

重新验证成功后：

1. 锁定目标构件；
2. 按固定优先级消耗材料；
3. 消耗1点机械电量；
4. 将加工动作加入动作队列；
5. 执行工序；
6. 工序牌继续留在加工台缓存中；
7. 被加工构件继续留在手牌中，除非工序导致其损坏、消耗或转化。

材料和电量必须在同一事务中支付。

不能出现：

* 已经消耗材料但没有执行加工；
* 已经消耗电量但目标失效；
* 工序执行两次；
* 同一张材料被两个加工台同时使用。

# 五、基础加工台取消自动连续加工

删除基础加工台原有的自动模式，包括：

* 自动连续检查；
* 自动登记固定工件；
* 自动重复加工同一张构件；
* 动作队列为空时自行启动。

新版基础加工台属于**半自动设备**：

* 自动寻找材料；
* 自动判断材料来源；
* 玩家手动启动；
* 玩家手动选择本次加工目标。

这样既减少机械槽位复杂度，也保留玩家选择需要培养哪张构件的决策。

未来若要实现自动加工，应通过独立能力、协议或高级机械实现，例如：

* 自动加工协议；
* 高级加工台的目标登记功能；
* 自动选择费用最高构件；
* 自动选择加工次数最多的构件。

不得把这些规则重新塞回基础加工台。

# 六、手工装配在加工台中的特殊逻辑

“手工装配”属于工序，可以放入加工台。

由于融锻需要主体和材料两张构件，其加工台执行方式为：

1. 启动时检查手牌中是否至少有两张合法构件；
2. 弹出第一层选择页面，让玩家选择融锻主体；
3. 再弹出第二层选择页面，让玩家从剩余合法构件中选择融锻材料；
4. 重新验证两张构件仍在手牌中；
5. 检查加工台电量；
6. 消耗1点机械电量；
7. 将材料构件融锻进主体；
8. 材料构件进入消耗区；
9. 主体构件留在手牌中；
10. 手工装配继续留在加工台工序缓存中。

手工装配不需要额外原料，其加工台材料要求为：

> 一张作为融锻材料的构件。

这里的融锻材料由第二层选择页面确定，不从仓储室自动获取。

# 七、所有产线机械统一启动失败播报

所有可主动或自动启动的机械都必须返回统一的启动检查结果。

不得让启动失败时没有反馈，也不得只在控制台输出错误。

建议建立统一结果结构：

```csharp
public enum MachineStartFailureReason
{
    None,
    NotPlayerTurn,
    ActionQueueBusy,
    MachineBusy,
    MachineDisabled,
    NoCachedOperation,
    InvalidCachedCard,
    NoLoadedCard,
    InsufficientCharge,
    MissingMaterial,
    MissingTarget,
    NoLegalCardTarget,
    HandFull,
    StorageFull,
    InvalidRegistration,
    AutoModeDisabled,
    TargetNoLongerValid,
    CardNoLongerAvailable,
    MachineSlotUnavailable,
    Unknown
}

public sealed record MachineStartResult(
    bool Success,
    MachineStartFailureReason FailureReason,
    string Message,
    int RequiredCharge = 0,
    int AvailableCharge = 0,
    string? MissingResourceName = null
);
```

每台机械启动前调用统一方法：

```csharp
MachineStartResult ValidateStart();
```

只有 `Success == true` 时，才能：

* 消耗电量；
* 消耗原料；
* 移动卡牌；
* 加入动作队列。

# 八、失败播报的表现

启动失败时至少同时提供两种反馈。

## 1. 机械模块状态文字

在机械模块内部短暂显示原因，例如：

* 未装入工序；
* 电量不足；
* 缺少机械原料；
* 没有合法构件；
* 手牌已满；
* 机械正在运行。

文字显示约1.5至2.5秒，然后恢复常态状态。

## 2. 战斗提示或浮动文字

在机械上方显示简短提示，例如：

```text
无法启动：未装入工序
无法启动：需要1点电量，当前0点
无法启动：缺少万能原料
无法启动：手牌中没有合法构件
无法启动：机械正在运行
```

不得只显示“启动失败”这类没有信息的通用文字。

## 3. 悬停详细说明

当机械不可启动时，启动按钮保持禁用或警告状态。

鼠标悬停启动按钮显示完整原因，包括：

* 缺少什么；
* 需要多少；
* 当前拥有多少；
* 哪个条件不合法。

---

# 九、失败原因优先级

若同时存在多个失败条件，只播报优先级最高的一项，避免一次出现大量提示。

加工台建议使用以下优先级：

1. 不是玩家回合；
2. 动作队列或选择界面忙；
3. 加工台正在运行；
4. 没有缓存工序牌；
5. 缓存牌不是合法工序；
6. 电量不足；
7. 缺少材料；
8. 手牌中没有合法构件；
9. 目标在确认前失效；
10. 未知错误。

例如加工台同时没有电量、没有材料和没有合法目标时，首先提示：

> 无法启动：电量不足。

获得电量后再次点击，才提示：

> 无法启动：缺少万能原料。

---

# 十、自动机械失败播报限制

自动机械不得在每次自动检查失败时不断重复播报。

自动检查失败的播报规则：

* 状态首次从“可启动”变为“不可启动”时播报一次；
* 失败原因发生变化时播报一次；
* 玩家主动点击启动时始终播报一次；
* 同一原因持续存在时不重复刷屏；
* 获得必要资源并恢复可启动后，清除旧失败状态。

例如自动启机协议控制的机械臂电量不足时：

* 首次尝试显示“电量不足”；
* 后续每帧检查不得重复显示；
* 获得电量后正常启动；
* 再次耗尽时可以重新显示一次。

# 十一、加工台 UI 修改

新版加工台模块只显示：

```text
加工台
电量：2/5

工序槽
[轻量化]

需要：
化合原料 ×1

[启动]
[取回工序]
```

删除：

* 工件槽显示；
* 投料槽显示；
* 固定加工对象显示；
* 自动模式按钮。

当玩家点击启动后，再通过独立的手牌选择页面选择构件。

## 启动按钮状态

### 可以启动

* 按钮正常高亮；
* 悬停提示当前工序和材料消耗。

### 没有工序

* 按钮禁用；
* 显示“未装入工序”。

### 缺少材料

* 按钮禁用或警告；
* 材料文字标红；
* 显示“缺少化合原料”。

### 电量不足

* 按钮禁用；
* 电量显示警告；
* 显示“需要1点电量”。

### 没有合法目标

* 按钮禁用；
* 显示“手牌中没有可加工构件”。

虽然按钮可以预先显示不可启动状态，玩家点击时仍应播报失败原因，除非游戏UI规范禁止点击禁用按钮。若完全禁用点击，则鼠标悬停必须提供完整失败原因。

# 十二、构件选择页面

建立通用的：

```text
FactoryCardSelectionOverlay
```

用于加工台和其他需要从手牌选择构件的机械。

需要支持：

* 单选；
* 多选；
* 有序多选；
* 合法目标过滤；
* 非法原因提示；
* 效果预览；
* 确认；
* 取消。

加工台普通工序使用单选模式。

手工装配使用有序双选模式：

```text
第一张：融锻主体
第二张：融锻材料
```

选择页面不得直接复制手牌卡牌实例，只显示对真实实例的引用。

关闭选择页面后，所有卡牌节点恢复原位置和交互状态。

# 十三、代码职责约束

## 加工台负责

* 保存工序牌；
* 检查电量；
* 请求材料解析；
* 请求合法目标列表；
* 打开选择页面；
* 将加工动作加入队列。

## 工序牌负责

* 声明合法目标规则；
* 声明所需材料；
* 声明具体加工效果；
* 声明是否允许重复；
* 声明目标不合法的原因。

## 材料解析器负责

* 搜索手牌；
* 搜索仓储室；
* 处理万能原料替代；
* 按固定顺序选择材料；
* 原子性消耗材料。

## UI负责

* 显示状态；
* 显示失败原因；
* 提供目标选择；
* 不直接执行加工逻辑；
* 不直接修改材料或电量。

# 十五、回归测试

至少实现以下测试。

## 加工台缓存

1. 只能放入带有工序的牌。
2. 工序牌可以免费取回。
3. 工序牌可以免费替换。
4. 手牌满时不能取回。
5. 不再存在工件槽和投料槽。

## 材料搜索

1. 优先消耗手牌材料。
2. 手牌没有时从仓储室消耗。
3. 优先专用原料。
4. 专用原料不足时使用万能原料。
5. 不从机械臂取料。
6. 不从分解室取料。
7. 多材料不足时不消耗任何材料。

## 启动流程

1. 没有工序时无法启动。
2. 电量不足时无法启动。
3. 材料不足时无法启动。
4. 没有合法构件时无法启动。
5. 所有条件满足时弹出手牌选择页面。
6. 取消选择不消耗任何资源。
7. 确认后重新验证。
8. 重新验证失败时不消耗任何资源。
9. 成功时只消耗一次电量和材料。
10. 工序牌执行后继续留在缓存中。

## 目标选择

1. 只显示手牌中的构件。
2. 不显示机械缓存构件。
3. 非法构件被过滤或置灰。
4. 固化只能选择一次性构件。
5. 标准升级不能选择已升级构件。
6. 手工装配正确进行有序双选。
7. 脆弱构件第二次加工时正确损坏。

## 失败播报

1. 每种失败原因都有明确文字。
2. 同时失败时只显示最高优先级原因。
3. 主动点击每次都能得到反馈。
4. 自动检查不会重复刷屏。
5. 失败不消耗电量。
6. 失败不消耗材料。
7. 失败不改变卡牌区域。

# 十六、最终验收标准

1. 基础加工台只保留一个工序缓存槽。
2. 不再保存固定工件。
